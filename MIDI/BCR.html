Creation / Class Methods


*new (deviceName, dict) create a BCR object.

deviceName - every MIDI port (e.g. from a USB plug) gets a unique ID. 

If you enter a device name, the class will loop through all available devices and connect to the right one (default "BCR2000").

dict - a dictionary of the actions that are mapped to controllers.

// simple example

b = BCR.new;

b.ccDict; // a dict where the maps will live

b.ccresp; // a midi CCResponder listing to that MIDI port

// map buttons to simple actions: 

b.mapCC('\btA1', { |ch, cc, val| " 'play': %.\n".postf([ch, cc, val]) });

// the actions are now in the dict:

b.ccDict.postcs;

b.free;


Class Variables

verbose

can be set to true to post info for debugging.


BCR.verbose = true;

BCR.verbose = false;

defaultCtlNames

a dictionary of names such as \knA1 or \btB3 to controller keys as used for lookup.

the keys are symbols like  '0_42' meaning midichannel 0, ctl num 42. 

// all names 

BCR.defaultCtlNames



Instance Methods

mapCC (ctl, action)


map one of the controls to a general function.


b = BCR.new;

b.mapCC('knC2', { |ch, cc, val| "3rd row of knobs, knob2: %.\n".postf([ch, cc, val]) });

b.mapCC('btA8', { |ch, cc, val| "1st row of buttons, button 8: %.\n".postf([ch, cc, val]) });

b.mapCC('tr1', { |ch, cc, val| "1st row of knobs, 1st knob, but we use trigger mode".postf([ch, cc, val]) });

b.ccDict;


mapTo (proxy, index, offset)


map all parameters from a NodeProxy.

1st row of knobs are mapped to the NodeProxy volume.

1st row of buttons: start-stop the NodeProxy.

the mapping is activated by pushing the corresponding button of the second row of buttons

if 


(

p = ProxySpace.push(s.boot);

~test = { |freq=250, intv=19, timescale=1, curve=0, loopnode=0, formfact=2, widfact=0.5, spread=1, amp=0.1| 

var env = EnvGen.kr(

Env(

{ 1.0.rand2 }!11, 

{1.0.rand}!10 * timescale, 

curve,

releaseNode: 9, 

loopNode: loopnode)

);

var pitch = (env * [1, 0.33, -1] * intv).midiratio * freq;

Splay.ar(

Formant.ar(pitch, pitch.scramble * formfact, pitch.scramble * widfact), spread) * amp;

};

// add specs for all the parameter ranges - required!

Spec.add(\intv, [0, 36, \lin]);

Spec.add(\timescale, [0.001, 10, \exp]);

Spec.add(\curve, [-10, 10]);

Spec.add(\loopnode, [0, 7, \lin, 1]);

Spec.add(\formfact, [0.1, 10, \exp]);

Spec.add(\widfact, [0.1, 10, \exp]);

Spec.add(\spread, \unipolar);

n = NodeProxyEditor(~test);

);


b = BCR.new; // make a new BCR

b.mapTo(~test, 1); // and map it to the index 1


// 1st knob of the 1st row is the volume

// 1st button underneath is on-off

// 1st button of the 2nd buttons row triggers the mapping (current params values are sent via midiout to the BCR so you can actually see their values)


// test using a second proxy, BCR index 2:

~test2 = ~test.source;

b.mapTo(~test2, 2);

// you can change this on the fly and assign an offset

b.mapTo(~test2, 3, \knC6)

// now toggle the 3rd button from the 2nd button row to see the changes on your BCR2000
