PocketDialKtl a class to use the midi controller Doepfer Pocket Dial with JITLIB.

inherits from MIDIKtl

IMPORTANT:

You need JITMIDIKtl quark to use PocketDialKtl.

See also: UsingMIDI, CCResponder, MIDIIn

1 - How to use

First of all, you might need to replace the midiCC values in the class itself if you use different ones than the defaults: PocketDialKtl::makeDefaults

I use midiCC 0, 1, … 15 for the bank 1, midiCC 16 … 31 for the bank 2 and so on until midiCC 63.

PocketDialKtl was first based on PDKtl (part of the quark JITMIDIKtl) but I use it differently (therefore this class).

Right now, PocketDialKtl uses the endless mode only. I use the preset 00101010 on my Doepfer Pocket Dial.

2 - Creation / Class methods

  • *new(srcID, ccDict)

    node - srcID - The src number may be the system UID (obtained from MIDIClient.sources[index].uid) or the index of the source in the MIDIClient.sources array. (nil matches all)

    node - ccDict - a Dictionary containing the controller elements names and their MIDI informations: (kn01: '1_0', kn02: '1_1', … )

    Example: 

    s.boot;
x = Ndef(\sin, { SinOsc.ar( \freq.kr(222), 0!2, \amp.kr(0.3) ) });
x.play;
d = PocketDialKtl.new
d.mapTo(x, 1) // mapped on the first knobs (bank 1)

4 - Instance methods

  • inform(boolean=true)

    post infos on params values 

    d.inform_(true)

  • mapTo(node, bank=1, offset=1, params=nil, stepmin=0.02, stepmax=0.5)

    node - a NodeProxy (like a Ndef or any proxy in ProxySpace)

    bank - the bank for midiCC (1 to 4)

    offset - the knob on which is assigned the 1st parameter (default: 1)

    params - an array of params to control (nil matches all)

    stepmin - the step used when a controller knob is turned slowly (default: 0.02)

    stepmax - the step used when a controller knob is turned fast (default: 0.5)

    maps a NodeProxy to some knobs automatically.

    Example: 

    s.boot;
x = Ndef(\sin, { SinOsc.ar( \freq.kr(222), 0!2, \amp.kr(0.3) ) });
x.play;
d = PocketDialKtl.new
d.mapTo(x, 1) // mapped on the first knobs (bank 1)

    By default the volume of the NodeProxy is mapped on the knob 16 of the bank.

  • mapVolume(node, ccnr, bank)

    node - a NodeProxy

    ccnr - the knob number (1 to 16)

    bank - the controller bank (1 to 4) 

    y = Ndef(\shwoing, { LeakDC.ar(LFPar.ar(SinOsc.kr(\freq.kr(80) + (..1), [0,pi], 400)))}).play
d.mapTo(y, 3, mapVol: false);
d.mapVolume(x, 1, 4); // map volume on kn01, bank 4
d.mapVolume(y, 2, 4); // map volume on kn02, bank 4

4.1 - Other options

  • specs - all parameters must have specs to be controlled 
    (
z = Ndef(\sinmod, { |fmod=12, mult=800, amp=0.3, dist=1|
    var snd = SinOsc.ar(SinOsc.kr(fmod, [0,pi], mult), 0, amp);
    (snd * dist).tanh * 0.3;
});
)
z.play;
d = d ?? PocketDialKtl.new;
// specs
(
Spec.add(\fmod, [10, 1e4, \exp]);
Spec.add(\mult, [400, 2000]);
// 'amp' already has a default spec
Spec.add(\dist, [1, 20]);
);
// map this one to the 2nd bank, starting on the 9th knob:
d.mapTo(z, 2, 9);

    Try moving the 9 10 11 12 knobs on your bank2.

  • params - choose which params to control (default all) 
    d.mapTo(z, 2, 4, [\freq, \dist]) // maps freq and dist only
  • steps

    We can also fine tune the step used by the controller: 

    d.mapTo(z, 2, 4, stepmin: 0.01, stepmax: 0.9);

    This will produce very small changes when the knobs are tweaked gently, and more drastic changes when turned fast.
